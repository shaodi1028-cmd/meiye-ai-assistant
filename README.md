# 美业 AI 运营助手

面向带轻医美服务的美甲美睫门店老板和员工的 AI 自媒体运营工作台。当前版本已经从静态原型升级为本地 Web 应用，包含 Node 后端和 JSON 本地数据存储。

## 运行方式

```bash
npm run dev
```

打开：

```text
http://localhost:4173
```

生产模式：

```bash
npm run start
```

Docker 运行：

```bash
docker compose up --build
```

重置演示数据：

```bash
npm run reset:data
```

健康检查：

```text
http://localhost:4173/healthz
```

## 真实模型测试

系统支持三种真实模型适配器，任选一种配置环境变量即可。都不配置时，会自动使用本地美业垂直模板。

```bash
OPENROUTER_API_KEY=你的_key npm run dev
```

或：

```bash
GEMINI_API_KEY=你的_key npm run dev
```

或：

```bash
GROQ_API_KEY=你的_key npm run dev
```

默认模型和服务配置可在 `.env.example` 查看。当前版本会自动读取项目根目录下的 `.env` 文件，也支持直接在终端或部署平台配置环境变量。

## 部署

部署说明见：


当前版本支持：

## 真实账号

登录页支持两种真实账号流程：

- 创建老板账号：填写门店名称、老板姓名、手机号和密码后进入工作台。
- 账号登录：老板和员工都可以用手机号和密码登录。

老板在“员工管理”里新增员工时，可以填写员工手机号和初始密码；填写后员工就可以用自己的账号进入员工视图。密码使用 PBKDF2 哈希后存储在 JSON 数据库的 `auth.accounts` 中，前端使用 session token 调用后端接口。

## 第一版产品边界

- 不自动发布到小红书、抖音、朋友圈
- 员工发布后手动提交链接或截图说明
- KPI 先按发布数量和完成率统计
- AI 生成逻辑暂用服务端模板模拟，后续可替换为真实模型 API

## 垂直模型设计

当前产品不是直接调用通用模型，而是先经过 `data/models/beauty-domain.json` 这层美业知识包：

- 服务识别：美甲、美睫、皮肤管理、轻医美
- 平台打法：小红书、抖音、朋友圈分别使用不同内容结构
- 专业规则：按项目卖点、内容角度、门店到店转化生成
- 合规规则：轻医美避免永久、根治、百分百、无风险等绝对化承诺
- 模型路由：文案、图片、视频分别走不同垂直模型类型

后续接入真实厂商时，底层可以换成任何大模型、图片模型或视频模型，但输出必须先通过这层美业垂直约束。

## 差异化生成

同一个项目、相似生成条件下，系统会根据员工、日期、项目、生成次数生成差异化策略：

- 标题钩子不同
- 内容角度不同
- 目标人群不同
- 使用场景不同
- 拍摄重点不同
- CTA 不同

生成结果里会显示“差异化生成策略”，方便老板判断员工之间是否撞稿。

## 老板审核

员工提交发布链接后，状态默认为待审核。老板可以在“内容审核”页面：

- 通过
- 退回
- 填写退回意见

## 项目素材库

当前本地数据在 `data/db.json` 的 `services` 字段中，老板也可以在网页里的“项目素材库”页面维护：

- 夏日显白冰透美甲套餐
- 自然裸感单根美睫
- 午休清洁补水皮肤管理
- 轻医美水光管理咨询

生成时选择项目素材后，系统会自动带入价格、护理时长、适合人群、卖点、注意事项和拍摄角度。

每个项目还支持更细字段：

- 禁忌提醒
- 护理后建议
- 加购/组合建议
- 素材关键词

## 门店内容素材库

老板可以在“内容素材库”页面维护案例和素材：

- 关联项目
- 关联员工
- 素材类型：案例图、科普卡片、视频片段、客户反馈
- 图片或素材链接
- 标签
- 使用说明
- 合规提醒

生成内容时，系统会自动引用同项目或同员工的素材，帮助 AI 生成更贴近真实门店的内容。

## 订阅与额度

当前支持演示套餐：

- 专业版试用：500 次/月
- 单店基础版：300 次/月
- 门店专业版：1000 次/月
- 连锁多店版：5000 次/月

每次生成内容会扣减一次额度。额度用完后，生成接口会返回提示，老板可以在“订阅与额度”页面切换套餐。

## 数据备份

老板可以在“数据备份”页面：

- 导出当前门店完整数据
- 复制备份 JSON
- 粘贴备份 JSON 后导入恢复
- 查看最近自动备份记录

后端每次写入 JSON 数据库前，会在 `MEIYE_BACKUP_DIR` 中保存一份自动备份，默认最多保留 10 份。

## 数据层

当前后端通过 `lib/json-store.js` 访问 JSON 数据库，业务代码只调用读写、导入、导出和备份方法。这样后续替换为正式数据库时，可以优先新增新的 store 实现，减少对业务 API 的影响。

如果 `MEIYE_DB_PATH` 指向的数据库文件不存在，系统会从 `MEIYE_DB_SEED_PATH` 自动复制一份初始数据。这个设计方便 Docker volume 或云平台持久化磁盘首次启动。

试用过程中如果创建了太多测试员工、素材或提交记录，可以执行 `npm run reset:data` 恢复到演示初始数据。系统会先把重置前数据保存到备份目录。
